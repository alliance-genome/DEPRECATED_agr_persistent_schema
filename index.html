Mixins

Slots

  • association slot - any slot that relates an association to another entity
    • negated - if set to true, then the association is negated i.e. is not true
    • object - connects an association to the object of the association. For example, in a gene-to-phenotype association, the gene is subject and phenotype is object.
    • predicate - A high-level grouping for the relationship type. AKA minimal predicate. This is analogous to category for nodes.
    • qualifiers - This is the MeSH qualifier term that is optionally added to the descriptor term.
    • relation - The relation which describes an association between a subject and an object in a more granular manner. Usually this is a term from Relation Ontology, but it can be any edge CURIE.
    • subject - connects an association to the subject of the association. For example, in a gene-to-phenotype association, the gene is subject and phenotype is object.
  • category
    • association➞category
  • data provider - MOD that provided data
  • date_produced - date AGR data generated by MOD
  • description - a human-readable description of an entity
  • id - A unique identifier for a thing. Must be either a CURIE shorthand for a URI or a complete URI
    • agent➞id - Different classes of agents have distinct preferred identifiers. For publishers, use the ISBN publisher code. See https://grp.isbn-international.org/ for publisher code lookups. For editors, authors and individual providers, use the individual's ORCID if available; Otherwise, a ScopusID, ResearchID or Google Scholar ID ('GSID') may be used if the author ORCID is unknown. Institutional agents could be identified by an International Standard Name Identifier ('ISNI') code.
    • resource➞id - Precedence of identifiers for references is as follows: PMID if available; DOI if not; actual alternate CURIE otherwise.
  • name - a human-readable name for an entity
    • agent➞name - it is recommended that an author's 'name' property be formatted as "surname, firstname initial."
  • node property - A grouping for any property that holds between a node and a value
    • address - the particulars of the place where someone or an organization is situated. For now, this slot is a simple text "blob" containing all relevant details of the given location for fitness of purpose. For the moment, this "address" can include other contact details such as email and phone number(?).
    • copyright date
    • creation date - date on which an entity was created. This can be applied to nodes or edges
    • information content entity➞creation date - Precedence of identifiers for references is as follows: PMID if available; DOI if not; actual alternate CURIE otherwise.
    • has authors
    • has biological sequence - connects a genomic feature to its sequence
    • has cross references
    • iso abbreviation
    • medline abbreviation
    • online issn
    • print issn
    • summary
    • symbol - Symbol for a particular thing
  • related to - A relationship that is asserted between two named things
    • from_species - holds between a genomic entity and a species
    • has assembly - holds between a genomic entity and an assembly
    • has editors - holds between a resource and a editor_resource
    • has feature - holds between a named thing and a feature
    • has gene - holds between a named thing and a gene
    • has genomic location - holds between a genomic entity and a genomic location
    • has go term - holds between a named thing and a gene ontology term
    • has identifier - holds between a named thing and an identifier
    • has note - holds between a named thing and a note
    • has ontology - holds between a named thing and an ontology
    • has publisher
    • has references - holds between a named thing and a reference
    • has so term - holds between a named thing and a sequence ontology term
    • has synonym - holds between a named thing and a synonym
    • has synonyms
    • has transcript - holds between a named thing and a transcript
    • on chromosome - holds between a genomic entity and a chromosome
  • release - MOD release ID
  • title - A human readable title for a reference.
    • resource➞title - the title of the publication
  • type - rdf:type of biolink:Association should be fixed at rdf:Statement
    • association➞type
  • volumes
