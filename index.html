Classes

  • Agent - person, group, organization or project that provides a piece of information (i.e. a knowledge association)
  • Assembly
  • AuthorReference
  • Entity - Root Biolink Model class for all things and informational relationships, real or imagined.
    • Association - A typed association between two entities, supported by evidence
    • ReferenceToMeshAssociation
    • CrossReference
    • NamedThing - a databased entity or concept/class
    • BiologicalEntity
      • MolecularEntity - A gene, gene product, small molecule or macromolecule (including protein complex)"
        • GenomicEntity - an entity that can either be directly located on a genome (gene, transcript, exon, regulatory region) or is encoded in a genome (protein)
          • Allele
          • Chromosome
          • Gene - A region (or regions) that includes all of the sequence elements necessary to encode a functional transcript. A gene locus may include regulatory regions, transcribed regions and/or other functional sequence regions.
          • Transcript - An RNA synthesized on a DNA or RNA template by an RNA polymerase.
          • Variant - Variant class
          • VariantProteinSequence - Class for variant protein sequences
    • MolecularConsequence - Class for molecular consequences calculated by the EnsEMBL Variant Effect Predictor (VEP)
      • GeneLevelConsequence - Class for gene-level VEP results
      • TranscriptLevelConsequence - Class for transcript-level VEP results
    • Ontology - a concept or class in an ontology, vocabulary or thesaurus
      • GoTerm
      • SoTerm

Slots

  • abstract
    • reference➞abstract - The abstract of the reference.
  • alliance category
    • reference➞alliance category - Controlled vocabulary term drawn from a list of Alliance-defined reference categories.
  • association slot - any slot that relates an association to another entity
    • negated - if set to true, then the association is negated i.e. is not true
    • object - connects an association to the object of the association. For example, in a gene-to-phenotype association, the gene is subject and phenotype is object.
    • reference to mesh association➞object - has topic is the relation used to describe the relationship between
    • predicate - A high-level grouping for the relationship type. AKA minimal predicate. This is analogous to category for nodes.
    • qualifiers - This is the MeSH qualifier term that is optionally added to the descriptor term.
    • relation - The relation which describes an association between a subject and an object in a more granular manner. Usually this is a term from Relation Ontology, but it can be any edge CURIE.
    • subject - connects an association to the subject of the association. For example, in a gene-to-phenotype association, the gene is subject and phenotype is object.
    • reference to mesh association➞subject
    • variant to entity association mixin➞subject - the variant that alters the molecular entity
  • category
    • association➞category
  • citation
    • reference➞citation - Is a reference to a source.
  • cnda end - End position of variant in cDNA coordinates
    • transcript level consequence➞cnda end
  • cns end - End position of variant in CDS coordinates
    • transcript level consequence➞cns end
  • data provider - MOD that provided data
    • variant➞data provider - Name of the MOD that submitted the variant data
  • date_produced - date AGR data generated by MOD
  • description - a human-readable description of an entity
  • display name
  • genomic variant sequence - variant genome sequence at variation position
    • variant➞genomic variant sequence - Alternative sequence at the variant position
  • id - A unique identifier for a thing. Must be either a CURIE shorthand for a URI or a complete URI
    • agent➞id - Different classes of agents have distinct preferred identifiers. For publishers, use the ISBN publisher code. See https://grp.isbn-international.org/ for publisher code lookups. For editors, authors and individual providers, use the individual's ORCID if available; Otherwise, a ScopusID, ResearchID or Google Scholar ID ('GSID') may be used if the author ORCID is unknown. Institutional agents could be identified by an International Standard Name Identifier ('ISNI') code.
    • reference➞id - the identifier for the publication in the form of a curie.
    • resource➞id - Precedence of identifiers for references is as follows: PMID if available; DOI if not; actual alternate CURIE otherwise.
    • variant➞id - Unique ID for variant
  • issue name
  • mod last modified date - The date the meta data on this object was last updated at the MOD.
    • reference➞mod last modified date
  • name - a human-readable name for an entity
    • agent➞name - it is recommended that an author's 'name' property be formatted as "surname, firstname initial."
  • node property - A grouping for any property that holds between a node and a value
    • address - the particulars of the place where someone or an organization is situated. For now, this slot is a simple text "blob" containing all relevant details of the given location for fitness of purpose. For the moment, this "address" can include other contact details such as email and phone number(?).
    • amino acid reference - reference genome amino acid sequence at variant position
    • transcript level consequence➞amino acid reference - Amino acid sequence encoded by codon(s) in reference genome sequence altered by the variant
    • amino acid variant - variant amino acid sequence at variant position
    • transcript level consequence➞amino acid variant - Amino acid sequence encoded by codon(s) in variant sequence
    • cdna end - end position of variation in cDNA coordinates
    • cdna start - start position of variation in cDNA coordinates
    • transcript level consequence➞cdna start - Start position of variant in cDNA coordinates
    • cds end - end position of variation in CDS coordinates
    • cds start - start position of variation in CDS coordinates
    • transcript level consequence➞cds start - Start position of variant in CDS coordinates
    • codon reference - reference sequence of codon(s) affected by variation - bases outside of the variant region are in lower case, those within are in upper case (e.g. cTa)
    • transcript level consequence➞codon reference - Reference genome sequence of codon(s) altered by the variant. Bases affected by the variant are given in upper case, bases flanking the variation are given in lower case
    • codon variant - variant sequence of codon(s) affected by variation - bases outside of the variant region are in lower case, those within are in upper case (e.g. cAa)
    • transcript level consequence➞codon variant - Sequence of codon(s) in variant sequence altered by the variant. Bases affected by the variant are given in upper case, bases flanking the variation are given in lower case
    • copyright date
    • corresponding author
    • creation date - date on which an entity was created. This can be applied to nodes or edges
    • information content entity➞creation date - Precedence of identifiers for references is as follows: PMID if available; DOI if not; actual alternate CURIE otherwise.
    • date arrived in pubmed - Day in which a reference or resource was created in PUBMED. Only relevant for PUBMED references.
    • reference➞date arrived in pubmed - As per the PUBMED XML record, this is the date the reference entered PUBMED.
    • date last modified - Date on which an entity was last modified by the MOD.
    • date last modified in pubmed - Date on which an entity was created.
    • date produced - Date on which an entity was produced.
    • date published - Date on which an entity was created.
    • reference➞date published - The date the reference was published. This field can be submitted by a MOD or from PUBMED. When pulled from PUBMED, this date is generated from the year, month and day published fields. When submitted from a MOD it can be any string. examples include: Spring 1991, 01-01-1991, 25-12-1991
    • day published - Day in which an entity was created.
    • reference➞day published - When the reference is a PUBMED reference, this will be a 2-digit day.
    • first name
    • genomic reference sequence - reference genome sequence at variation position
    • variant➞genomic reference sequence - Reference sequence at the variant position

    • reference➞has tags - Tags from the MOD from a controlled vocabulary of possible tags.

    • transcript level consequence➞hgvs protein nomenclature - HGVS protein sequence (HGVSp) name

    • polyphen prediction - PolyPhen-2 prediction

    • molecular consequence➞polyphen score - Pathogenicity score, between 0 and 1, calculated by PolyPhen-2
    • print issn
    • protein end - end position of variation in protein amino acid coordinates
    • transcript level consequence➞protein end - End position of variant in amino acid sequence
    • protein start - start position of variation in protein amino acid coordinates

    • year published - Year in which an entity was created.
    • reference➞year published - When the reference is a PUBMED reference, this will be a 4-digit year.
  • padding left - flanking sequence upstream of the variation position on the sense strand of the reference genome
